5. 作业总结
	1. 核心概念：输入永远是“字” (Strings)
		1. 函数就像一个只会收信的门卫。哪怕用户输入的是数字 `50`，它收进来的也是一张画着“50”的纸条
		2. **类型转换 (Casting)：** 如果你要做数学运算，必须手动把“字”变成“数”
		3. 对比 C 语言：
			1.  C：`scanf("%d", &n)` —— 进门时就严查身份证，不是数字不准进。 
			2. Python：`input()` —— 谁都能进，进来后再由你决定给它穿什么衣服（转换类型）。
	2. 字符串是“不粘锅”
		1. 字符串一旦创建，就**不可修改**。
		2. 当你使用 `.lower()` 或 `.replace()` 时，Python **并没有**修改原来的变量，而是**制造了一个新的副本**。
		3. 你必须用变量去“接住”这个新副本（比如 `new_s = s.replace(...)`），否则它就会凭空消失。
	3.  工具箱：方法 (Methods) 与 格式化
	    - **内置外挂：** Python 给对象（比如字符串）自带了很多工具
        - 变小写：`.lower()`
        - 去空格：`.strip()
        - 替换：`.replace()`
        - 首字母大写：`.title()`
	    - **魔法填充 (f-string)：** `f"E: {e}"`。这是 Python 最强大的输出方式，直接把变量的值“嵌入”到字符串里，不用像 C 语言那样写 `%d` 还要数位置。
     4. 函数的职责 (Functions)
	     1. **定义函数 (`def`)：** 用来封装一段逻辑。
	     2. **Return vs Print：** 这是初学者最容易混淆的。
		     1. `return`：像工厂交货。把处理好的数据扔回去给调用者（主程序），**不显示**在屏幕上。
		     2. `print`：像拿大喇叭喊。直接显示在屏幕上，但程序内部拿不到这个数据。
		 3. 在 _Tip Calculator_ 中，你学会了只修补特定功能的函数（`dollars_to_float`），而不去动主程序 (`main`)，这是“模块化编程”的雏形。
	5. C vs Python 的思维差异
		1. **C 语言 (左侧思维)：** 你是**内存管理员**。你需要知道每个数据占几个格子，字符数组怎么移动，怎么用指针去指每一个字母。
		2. **Python (右侧思维)：** 你是**流水线主管**。你不需要知道数据在内存里长什么样，你只需要下达指令：“把这箱原料拿去‘去空格’车间，再拿去‘变小写’车间，最后拿去‘转数字’车间。”
